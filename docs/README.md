
# Project Documentation

This `docs` folder provides comprehensive documentation for the QE4SE Catalyst project. Below is an overview of its structure and the purpose of each file and subfolder:

## Main Documentation Files

- **AI_DEVELOPMENT_GUIDE.md**: Best practices and guidelines for AI development in this project.
- **architecture.md**: High-level overview of the project's architecture.
- **coding-standards.md**: Coding standards and conventions to ensure code quality and consistency.
- **development-guide.md**: General development workflow, environment setup, and contribution guidelines.
- **testing-practices.md**: Strategies, tools, and standards for testing across the project.

## Subfolders

- **backend/**
	- `architecture.md`: Backend architecture details.
	- `coding-standards.md`: Backend coding standards.
	- `development-guide.md`: Backend development workflow and setup.
	- `testing-practices.md`: Backend testing strategies and practices.

- **frontend/**
	- `architecture.md`: Frontend architecture details.
	- `coding-standards.md`: Frontend coding standards.
	- `development-guide.md`: Frontend development workflow and setup.
	- `testing-practices.md`: Frontend testing strategies and practices.

---

Refer to the relevant files and folders above for detailed guides, standards, and architectural information for both backend and frontend development.
